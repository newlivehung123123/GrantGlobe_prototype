## PHASE 0 — PROMPT 1: Database Export Script

**What this builds**
A Python script that connects to the Stage 3 PostgreSQL database and exports
quality-assured grant records to `Stage_4_interface/data/grants.json`. This script
is re-run each time the interface data needs to be refreshed (e.g. after each weekly
crawl-and-extract cycle).

**Read this before building.**

The export must only include records that are ready for public display. The
inclusion rule is:

```
(review_status = 'approved')
OR
(requires_review = false AND review_status = 'pending')
```

Excluded: `review_status = 'rejected'` and records where `requires_review = true`
AND `review_status = 'pending'` (flagged, awaiting human review).

This means high-confidence records appear automatically; low-confidence records
only appear after a human approves them.

---

**Create `Stage_4_interface/export_grants.py`**

```python
"""
GrantGlobe Stage 4 — Database Export Script

Exports quality-assured grant records from the Stage 3 PostgreSQL database
to data/grants.json for use by the static interface.

Usage:
    python export_grants.py
    python export_grants.py --output data/grants.json   # default path
    python export_grants.py --include-closed            # include Closed grants

Reads DATABASE_URL from the environment (or .env file in Stage_3_LLM_extraction/).
"""
```

**Implementation requirements:**

1. Load `DATABASE_URL` from environment. If not set, print a clear error message
   and exit with code 1.

2. Query the `grants` table with the inclusion rule above. Order by:
   - `current_status` priority: Open first, then Upcoming, then Rolling, then others, then Closed last.
   - Within each status group: `application_deadline ASC NULLS LAST`.

3. By default, exclude `current_status = 'Closed'` grants from the export unless
   `--include-closed` flag is passed. Closed grants inflate the dataset without
   helping users who want active opportunities.

4. Serialise every field for JSON:
   - `DATE` columns → ISO 8601 string (`"2026-12-31"`) or `null`
   - `TIMESTAMPTZ` columns → ISO 8601 string or `null`
   - `NUMERIC` columns → Python `float` or `null`
   - `TEXT[]` / `CHAR[]` arrays → Python list (never `null` — use `[]` if null in DB)
   - `BOOLEAN` → Python `bool` or `null`
   - `JSONB` columns (`confidence_scores`, `raw_extraction`) → **exclude from export**
     (internal quality data, not needed in the interface)
   - `UUID` → string

5. Output JSON structure:

```json
{
  "metadata": {
    "exported_at": "2026-05-28T10:00:00Z",
    "total_grants": 847,
    "schema_version": "1.0",
    "includes_closed": false
  },
  "grants": [
    {
      "id": "uuid-string",
      "grant_title": "Innovation Research Fund",
      "funder_name": "Wellcome Trust",
      "source_url": "https://wellcome.org/grants/innovation",
      "application_portal_url": null,
      "description": "Funds early-stage innovation research...",
      "application_deadline": "2026-12-31",
      "application_deadline_raw": "31 December 2026",
      "application_deadline_type": "confirmed",
      "deadline_notes": null,
      "eoi_deadline": null,
      "eoi_deadline_raw": null,
      "grant_opening_date": null,
      "funding_amount_min": 10000.0,
      "funding_amount_max": 50000.0,
      "currency": "GBP",
      "current_status": "Open",
      "source_language": "en",
      "ai_focused": false,
      "individuals_not_eligible": false,
      "organisation_types": ["University", "NGO"],
      "individual_eligibility": ["Postdoctoral researcher"],
      "applicant_base_regions": ["Global"],
      "applicant_base_countries": [],
      "geographic_focus_regions": ["Sub-Saharan Africa", "South Asia"],
      "geographic_focus_countries": ["KE", "IN", "UG"],
      "thematic_sectors": ["Health and Medical Research"],
      "grant_types": ["Research Grant"],
      "domain": "wellcome.org",
      "crawl_date": "2026-05-23"
    }
  ]
}
```

6. Write the output file with `ensure_ascii=False` and `indent=2`.

7. Print a summary on completion:
```
Export complete.
  Records exported:  847
  Includes closed:   No
  Output:            data/grants.json
  Exported at:       2026-05-28T10:00:00Z
```

**`--output` flag:** allow overriding the output path (default: `data/grants.json`).
Create the output directory if it does not exist.

---

**Acceptance criteria**
- `python export_grants.py` runs without error against the Stage 3 database.
- `data/grants.json` is created with valid JSON (verify: `python -c "import json; d=json.load(open('data/grants.json')); print(d['metadata']['total_grants'], 'grants')`).
- All `DATE` fields are strings or null (no Python `date` objects).
- All array fields are lists (no nulls).
- `confidence_scores` and `raw_extraction` are absent from every grant object.
- `python export_grants.py --include-closed` includes Closed grants and sets
  `metadata.includes_closed = true`.

**`renderCards(grants)`**

Renders grant cards into `#grants-list`. Each card must show:

- **Header row**: grant title (left) + status badge (right)
- **Funder line**: funder name, prefixed with a building icon (🏛)
- **Meta row**: up to three meta items shown as `icon + text`:
  - Deadline: 📅 formatted as "31 Dec 2026" (or "Rolling" / "TBC" if applicable)
  - Funding amount: 💰 "£10,000 – £50,000 GBP" (skip if both min and max are null)
  - Language: 🌐 (only if `source_language` is not "en")
- **Tags row**: geographic focus regions (first 2) + thematic sectors (first 2),
  each as a `.tag` element. If more than 4 total tags: append "+N more" tag.
- **Clicking anywhere on the card** opens the detail modal for that grant.
- Cards with `current_status = 'Closed'` receive an additional CSS class
  `grant-card--closed` that sets `opacity: 0.65`.

**Deadline formatting rule:**
- `application_deadline_type = 'rolling'` → show "Rolling"
- `application_deadline_type = 'tbc'` → show "TBC"
- `application_deadline_type = 'not_published'` → show "Not published"
- `application_deadline` is a valid date string → format as "31 Dec 2026"
- Otherwise → show "—"

**Funding amount formatting rule:**
- Both null → omit the meta item entirely
- Only min present → "From £10,000"
- Only max present → "Up to £50,000"
- Both present, equal → "£50,000"
- Both present, different → "£10,000 – £50,000"
- Always append currency code: "£10,000 – £50,000 GBP"
- Currency symbol mapping: GBP→£, USD→$, EUR→€, others→display the code

**`openModal(grant)`**

Populates `#modal-body` with the full grant detail and shows the modal overlay.

Modal sections (only render a section if the value is non-null and non-empty):

1. Title + funder + status badge (always shown)
2. **At a glance** table:
   - Application deadline (formatted as above)
   - EOI deadline (if present)
   - Grant opening date (if present)
   - Funding amount (if present)
   - Currency
   - AI focused: Yes / No (always shown)
   - Open to individuals: Yes / No (based on `individuals_not_eligible`)
3. **Description** (if present, full text)
4. **Geographic focus** — regions as tags + country codes as tags
5. **Applicant base** — regions and countries
6. **Thematic sectors** — as tags
7. **Organisation types** — as tags
8. **Individual eligibility** — as tags
9. **Links**: "View original grant page" (source_url), "Apply here" (application_portal_url if different)
10. **Provenance** (small, muted): "Source: {domain} · Crawled {crawl_date}"

Close modal on: close button click, overlay click (outside modal content), Escape key.
When modal opens: trap focus inside modal. When modal closes: return focus to the card that opened it.

```javascript
      { name: "description",   weight: 1 },
      { name: "thematic_sectors", weight: 1 },
    ],
    threshold: 0.35,      // lower = stricter match
    includeScore: true,
    minMatchCharLength: 2,
  });
}
```

**`applySearchAndFilters()`**

The central function called whenever search query or any filter changes.
Returns the filtered, sorted array and re-renders the page.

Logic (apply in this order):

1. **Search**: if `state.searchQuery.length >= 2`, use Fuse.js to get matching
   grants (extract the `item` from each result). If query is empty or < 2 chars,
   use `state.allGrants` directly.

2. **Status filter**: if `state.activeFilters.status.length > 0`, keep only grants
   whose `current_status` is in the selected set.

3. **Region filter**: if `state.activeFilters.regions.length > 0`, keep only grants
   where `geographic_focus_regions` contains AT LEAST ONE of the selected regions.

4. **Sector filter**: if `state.activeFilters.sectors.length > 0`, keep only grants
   where `thematic_sectors` contains AT LEAST ONE of the selected sectors.

5. **AI focused filter**: if `true`, keep only grants where `ai_focused === true`.

6. **Individuals eligible filter**: if `true`, keep only grants where
   `individuals_not_eligible === false`.

7. **Sort**:
   - `deadline_asc`: Open/Upcoming grants sorted by `application_deadline` ascending
     (nulls last), then Rolling, then Closed.
   - `deadline_desc`: reverse deadline order.
   - `title_asc`: alphabetical by `grant_title`.
   - `funder_asc`: alphabetical by `funder_name`.

8. Update `state.filtered`, call `renderCards(state.filtered)`, update results count.

**`updateResultsCount(n)`**

Updates `#results-count` text:
- `"1 grant found"` / `"847 grants found"` / `"No grants found"`

**Dynamic filter population**

After loading grants, populate `#filter-regions` and `#filter-sectors` with checkboxes
built from the unique values across all grants:

- Regions: collect all unique strings from `geographic_focus_regions` across all
  grants, sort alphabetically, render as checkboxes.
- Sectors: collect all unique strings from `thematic_sectors` across all grants,
  sort alphabetically, render as checkboxes.
- Each checkbox uses the same `.filter-option` pattern as the static checkboxes in
  the HTML.

**Event listeners (wire up all controls):**

- `#search-input` `input` event → update `state.searchQuery`, call
  `applySearchAndFilters()` (debounced 200ms)
- All filter checkboxes `change` event → update `state.activeFilters`, call
  `applySearchAndFilters()`
- `#sort-select` `change` event → update `state.sortBy`, call
  `applySearchAndFilters()`
- `#clear-filters` and `#empty-clear-filters` `click` → reset all filters,
  clear search input, call `applySearchAndFilters()`

**Header metadata**

After loading, populate `#header-meta` with:
```
847 grants · Last updated 28 May 2026
```
Parsed from `metadata.total_grants` and `metadata.exported_at`.

**Footer date**

Set `#footer-date` to the `exported_at` date formatted as "28 May 2026".
